# AI 选品助手 AI 参与开发协议 v0.1

## 目标

> 允许开发者使用不同 AI 工具，但所有产出必须遵守同一套项目协议、边界和验证规则。

不统一模型，不统一提示词；统一的是：**输入边界、输出格式、验证流程和合并标准**。

## 项目公约

- **协议先于代码**
- **人负责，AI 辅助**
- **结果统一，工具自由**
- **局部内容不得污染全局规则**
- **所有 AI 产出默认是候选草稿，不是最终事实**

## AI 可以参与

- 需求分析与方案拆解
- 文档草稿、协议草稿
- 爬虫脚本草稿
- AI 分析逻辑草稿
- 文案生成模板
- 测试用例草稿
- 重构建议与 Review 辅助

## AI 不能替代

- Schema 最终裁定
- 架构边界决策
- 合并审批
- 对平台规则与合规性的判断
- 对隐含需求的主观扩写

一句话：**AI 是协作者，不是决策者。**

## 协议源头

AI 参与开发时至少必须遵守：

- `README.md`
- `CONTRIBUTING.md`
- `docs/DATA_SCHEMA.md`
- `docs/ARCHITECTURE.md`

如果 AI 输出与这些文档冲突，以文档为准。

## 统一开发约束

### 1. 统一命名

AI 生成内容必须遵守项目既有命名，不允许：

- 同一概念多个名字（如 `product` / `item` / `goods` 混用）
- 未定义的中英文混用
- AI 自创缩写
- 自发引入新的字段体系

### 2. 统一目录

| 产出类型 | 目录 |
|---------|------|
| 文档 | `docs/` |
| 爬虫 | `scraper/` |
| 分析 | `analyzer/` |
| 货源匹配 | `matcher/` |
| 文案生成 | `copywriter/` |
| 看板 | `dashboard/` |
| 测试 | `tests/` |
| 本地数据 | `data/` |

### 3. 统一 Schema

AI 不允许：

- 擅自新增字段
- 擅自更改字段类型
- 擅自更改枚举语义
- 擅自更改必填/可选含义

需要改 Schema 必须先走协议变更流程（提交 claim + 单独 PR）。

## 核心工作模式：Task First

正确顺序：

1. 从任务列表选任务或补任务（`docs/AI_SHARED_TASKLIST.md`）
2. 中高风险任务先提交模块认领说明（`docs/claims/`）
3. 给 AI 明确边界
4. AI 辅助产出
5. 人工验证后提交

### 每个任务卡至少包含

- 目标
- 范围
- 输入协议
- 输出物
- 不可触碰边界
- 验收标准
- 验证方式

## 每次 AI 参与必须带的输入声明

调用 AI 前至少声明：

- 当前任务目标
- 允许修改哪些文件
- 不允许修改哪些文件
- 依据哪些协议文档
- 哪些内容不能假设
- 必须完成哪些验证

没有边界声明的 AI 参与，视为不合规。

## AI 产出 6 条硬规则

1. **不可修改无关文件**：只能改任务允许范围内的文件
2. **不可虚构依赖**：不允许引用不存在的模块、接口或数据
3. **不可绕过协议**：协议未定义内容，AI 不能自行拍板落地
4. **不可跳过验证**：按改动类型选择 lint / test / schema validate / smoke test
5. **不可省略变更说明**：每次 commit 必须在 `docs/changes/` 留文档
6. **不可把 AI 输出视为真相**：AI 输出默认只是候选实现

## 风险等级

### A 类（低风险）——AI 高参与

- 文档整理
- 测试补全
- 小工具脚本
- 文案模板调整

### B 类（中风险）——AI 辅助，人主导

- 模块内爬虫逻辑
- 分析规则实现
- 数据转换流程
- 看板局部功能

### C 类（高风险）——AI 只辅助

- Schema 变更
- 核心架构变更
- 跨模块联动重构
- 全局命名调整

## PR 要求

如有 AI 参与，PR 必须说明：

- AI 参与了哪个环节（分析/草稿/实现/测试/文档）
- 依据了哪些协议文档
- 对应的认领说明在哪里（如适用）
- 对应的 commit 文档说明在哪里
- 做了哪些人工确认
- 跑了哪些验证

## Reviewer 检查重点

- 是否越界修改
- 是否偷偷改 Schema
- 是否新增未定义字段或枚举
- 是否伪造依赖
- 是否完成验证
- 是否遵守认领说明边界

## 明确禁止的行为

- 直接让 AI 自由生成后原样提交
- 未定义协议就先写实现
- 在功能 PR 中夹带 Schema 修改
- AI 擅自新增字段、枚举、目录结构
- 提交未经验证的 AI 代码
- 未认领就开展中高风险模块开发
- 提交没有附带 `docs/changes/` 说明的 commit
- 提交"我没看懂但 AI 说可以"的内容
