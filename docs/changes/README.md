# 变更说明目录

`docs/changes/` 记录每次实质性 commit 的变更说明。

## 什么时候必须写

每次实质性提交必须在此目录新增一份说明文件。

可豁免：typo、纯格式调整、小范围文档润色。

## 文件命名

格式：`<date>-<slug>.md`

示例：
- `2026-03-15-scraper-xianyu-v1.md`
- `2026-03-15-schema-add-product-score.md`

## 内容模板

```md
# 变更说明：<标题>

- 日期：
- 模块：
- 改动类型：协议 / 功能 / 数据 / 文档 / 测试
- 是否涉及 Schema 变更：是 / 否

## 为什么改

## 改了什么

## 影响的文件或模块

## 没改什么

## 验证方式与结果

## 风险点
```

## 注意

变更说明记录「已发生的事实」，不是计划书。先完成实现与验证，再补说明。
